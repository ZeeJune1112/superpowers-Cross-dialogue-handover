# 2048 Game Design Spec

## Overview

Standard 2048 puzzle game implemented in Python with pygame, packaged as a standalone Windows .exe via PyInstaller.

## Requirements

- 4x4 grid, arrow key controls
- Random tile spawning (2 or 4) after each move
- Tile merging with score accumulation
- Game over detection (no valid moves remaining)
- Win detection (2048 tile reached, with option to continue)
- Restart via R key
- Single .exe output via PyInstaller

## Tech Stack

- Python 3
- pygame (rendering + input)
- PyInstaller (packaging)

## Project Structure

```
2048/
  game.py            # Main game (logic + render + input in one file)
  requirements.txt   # pygame, pyinstaller
  build.bat          # One-click build script
```

## Architecture

### Game Board (Model)

- 4x4 grid stored as `list[list[int]]`, 0 = empty
- `move(direction)` — slide and merge tiles in one of 4 directions (up/down/left/right)
  - Slide: shift all non-zero tiles toward the target edge
  - Merge: combine adjacent equal tiles, score += merged value
  - Returns whether the grid changed
- `spawn()` — place a 2 (90%) or 4 (10%) in a random empty cell
- `is_game_over()` — no empty cells and no adjacent equal tiles
- `has_won()` — any tile >= 2048

### Renderer (View)

- pygame window: 500x600 (500x500 grid + 100px score panel)
- Grid: dark background with cell borders
- Tiles: colored rectangles with centered white/black numbers
- Standard 2048 color scheme per tile value
- Score display at top
- "Game Over" overlay with restart prompt
- "You Win!" overlay with continue option

### Input (Controller)

- Arrow keys → `move(direction)`
- R → restart (re-initialize grid)
- ESC / window close → quit

### Main Loop

```
while running:
    handle events (input)
    update game state
    render frame
    clock.tick(60)
```

## Data Flow

```
ArrowKey → move(dir) → grid changes? → spawn() → check game_over/win → render
```

## Build & Output

```batch
pip install -r requirements.txt
pyinstaller --onefile --windowed --name 2048 game.py
```

Output: `dist/2048.exe`

## Scope Boundaries

**In scope:** core gameplay, rendering, input, single .exe packaging
**Out of scope:** animations, sound, undo, save/load, settings menu, mobile support
